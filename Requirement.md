# サービス要件

## サービス概要\/前提

- 書籍で出版するガイドブックのデジタル/ウェブ版として制作する
- 書籍の形でパッケージしている建築の見方とは異なるしかたで提供する

## 用途

- 建築を閲覧する
  - マップ上から配置や経路などを見る
  - 建築の詳細情報を確認する
- 建築をめぐるためのルートを提案、提示する

## 実装機能

### 基本機能

- マップ表示
  - マップピン表示
  - 経緯度情報を任意のファイルから読み込み、マップ上にピンを落とす
  - 現在地から近いピン情報(=他の建築)をサジェスト
  - 現在地が移動された場合に、都度更新
  - おすすめする建築の近さの最高値、または現在地から他の建築を探すエリア半径を下辺にできるようにする。
- 情報表示
  - ピンされている建築の詳細情報の表示
  - 基本情報
    - 建物タイトル
    - 設計者
    - 用途
    - 竣工年
    - 住所
    - 写真
  - 訪れた建物のリスト機能
  - GPSで特定の建物の近傍に行くと、アプリ側で行った記録を保存。（アチーブメント）
- Basic認証
  - テストのため簡易的なもので良い

## スタイル

### 定義

#### マップの中心

- PC
  - x：各種情報のボックス（パネル）の右端から画面の右端の中心
  - y：マップ画面の縦幅の中心
- スマートフォン / タブレット
  - マップ表示エリアの中心

#### マップピン

- 現在地
  - 緑
- 建築
  - 表示時：青
  - 選択時：赤
- ルート表示の出発地・目的地・経由地
  - 赤
- お気に入り
  - ピンク

### 全体の要素

- マップ
  - 現在地ボタン
  - 建築作品のピン情報
- サイドパネル（PC） / 情報表示パネル（スマートフォン・タブレット）
  - 建築情報の詳細情報
    - 選択したピンの建築の詳細情報を表示するパネル
  - 近くの建築物（NEAR BY）
    - 現在地の近くにある建築作品を近い順・リストで表示
  - アチーブメント（Archivement）
    - 訪れた建築作品を記録する

#### 各部について

- サイドパネル（PC） / 情報表示パネル（スマートフォン・タブレット）
  - 表示項目
    - 画像
      - 複数の場合あり
      - 複数の場合：矢印キーなどで順番に1枚ずつ表示
      - 画像がない場合：`noimage.jpg`を表示
    - 建物タイトル（Title）
    - 設計者名（Architect）
    - 竣工年（Completion）
    - 建築用途（Building Use）
      - 複数ある場合、カンマ繋ぎで表記
    - 地域（Region）
    - 住所情報（Address）
    - キャプション
    - タグ
    - 関連リンク（More Information）
    - ルートボタン
    - お気に入りボタン
  - 近くの建築物（NEAR BY）
    - 建物ごとに下記要素
      - 現在地からの距離
      - ルートボタン
      - お気に入りボタン
  - お気に入り建築リスト
    - お気に入りに登録した建物をリストで表示
  - アチーブメント
    - 訪れた建築作品をリストで表示
    - お気に入りボタン
    - 畳めるようにする

### PC

#### 画面構成

- マップを全面に展開する
  - ピンを配置
  - 現在地を表示
- サイドパネル
  - 幅：画面サイズの30%
  - 高：サイドパネル要素を並べた高さ
- 現在地機能
  - 現在地ボタンを画面右下に表示

#### 挙動

- マップ
  - 現在地ボタン・作品マップピンを選択すると*PCでの画面中央*にピンの位置を合わせる
  - ルートを表示した際、ルート全体が表示できるように画面を移動する。
    - ルートの開始ピンと終了ピンの中心がPCでの画面中央に位置する
    - ルートの開始ピンと終了ピンがサイドパネルを除いた画面範囲に収まる
- サイドパネル全体
  - 建築詳細情報、近くの建築パネルに配置されているルートボタンを押すとルートを計算し、表示する。
  - 情報がないパネルについては畳めるようにする。
  - サイドパネル部分のみでスクロール可能
    - サイドパネル高さが画面高さを超える場合、スクロールして全体が確認できるようにする。
- 建築詳細
  - ピンが選択されていない場合畳む、あるいは非表示にする
  - ピンを選択すると表示する
  - タグを押すと、マップ上のピンがそのタグがついたものだけに抽出・表示される
- お気に入りボタン
  - 建築詳細、近くの建築、アチーブメントパネルの右側、作品ごとにハートアイコンを設置。
  - 押すと登録
    - 空洞のハートが登録すると中身がフィルされる

### スマートフォン\/タブレット

#### 画面構成

- マップを画面全体に展開する
- 画面右下に現在地ボタンを表示する
- 画面高さの1/5にタブをマップに重ねるように表示する
  - `Tap for Detail`をタブ上部に配置
  - `Tap for Detail`をタップすると画面全体に建築詳細画面を展開
- 画面左上に「近くの建築」「アチーブメント」「お気に入り作品」をボタンで表示
  - ボタンを押すとそれぞれの画面が表示される
  - 画面の4/5程度のサイズ
  - スライド表示された画面の右上に`×`アイコンを表示

#### 挙動

- マップ
- 現在地ボタン・作品マップピンを選択すると*スマートフォン / タブレットでの画面中央*にピンの位置を合わせる
- ルートを表示した際、ルート全体が表示できるように画面を移動する。
  - ルートの開始ピンと終了ピンの中心がスマートフォン / タブレットでの画面中央に位置する
  - ルートの開始ピンと終了ピンがサイドパネルを除いた画面範囲に収まる
- 建築詳細
  - 画面縦幅の1/5程度で常時表示
    - タイトル
    - お気に入りボタン＋ルートボタン
  - `Tap for Detail`をタブ上部に配置
  - `Tap for Detail`をタップすると画面全体に建築詳細画面を展開
- 近くの建築 / アチーブメント共通
  - ボタンを押すと左側からスライドして表示
    - スライド表示の際に「建築詳細」タブを一時的に非表示にする
    - 近くの建築 / アチーブメントを消すと再度表示する
  - 表示画面の右上`×`アイコンをタップするとスライドして非表示になる
- アチーブメント
  - PCのGPS精度限界（5〜10m）で建物の接近を検知
  - 接近を検知すると、`Visit`ボタンを画面上部に表示する。
    - ボタンを押すとアチーブメントに追加される

## 取り扱い情報

- JSON
  - `./data`に配置
  - `init.json`
    - ピンを落とすデータを想定して作成
    - タイトル・設計者・経緯度
  - `main.json`
    - 建築ごとのページなどの表示を想定して作成
    - 詳細情報全て
    - `builduse`と`tag`は配列でデータを保持。
      - 建築用途：建築用途の表示に使用。複数の用途を同時に持つこともある。
      - タグ（`tags`）：タグ検索・表示などに使用する想定。
  - `photo.json`
    - タイトル・設計者とともにGCSに格納された画像のリンクを格納している
- 画像
  - `photo.json`で参照できるGCSのリンクを使用する
