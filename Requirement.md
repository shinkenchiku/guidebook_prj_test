# サービス要件

## サービス概要

- ガイドブックのデジタル版として制作
- マップなどを用い、建築やルートを案内するサービス

## 実装したい機能

- マップのピンを表示
  - 経緯度情報を任意のファイルから読み込み、マップ上にピンを落とす
- 現在地から近いピン情報(=他の建築)をサジェスト
  - 現在地が移動された場合に、都度更新
  - おすすめする建築の近さの最高値、または現在地から他の建築を探すエリア半径を下辺にできるようにする。
- ピンされている建築の詳細情報の表示
  - 基本情報
    - 建物タイトル
    - 設計者
    - 竣工年
    - 住所
    - 写真
- 訪れた建物のリスト機能
  - GPSで特定の建物の近傍に行くと、アプリ側で行った記録を保存。（アチーブメント）
- Basic認証
  - テストのため簡易的なもので良い

## スタイル

### 定義

- マップの中心
  - PC
    - x：各種情報のボックスの右端から画面の右端の中心
    - y：マップ画面の縦幅の中心
  - スマートフォン / タブレット
    - マップ表示エリアの中心
- マップピン
  - 現在地
    - 緑
  - 建築
    - 表示時：青
    - 選択時：赤
  - ルート表示の出発地・目的地・経由地
    - 赤

## 全体

- 構成
  - マップ
    - 現在地表示＋現在地ボタン
      - 建築詳細画面が出ている時は、その分上に移動する
    - 作品のピン情報
  - タイトルなど
  - 情報表示
    - 建築情報の詳細
      - マップ上のピンを押すと表示
      - 画像
        - 複数の場合あり
        - 複数の場合：矢印キーなどで順番に1枚ずつ表示
        - 画像がない場合：`noimage.jpg`を表示
      - 建物タイトル
      - 設計者名
      - 竣工年（Completion）
      - 建築用途（Building Use）
      - 地域（Region）
      - 住所情報（Address）
      - キャプション
      - タグ
      - 関連リンク（More Information）
    - 近くの建築物（Nearby）
      - 現在地から指定距離範囲内の建物のリスト
      - 近い順
    - アチーブメント
      - 訪れた建築のリスト
      - 現在地が建物ピンの位置の半径1メートル程度で接近した場合に追加される

### PC

- 画面全体
  - マップ画面を全面に展開する
  - マップの上に各種情報を配置する
    - 左寄せ、画面横幅の1/3程度の横幅
    - 要素
      - ロゴ（今後データが追加できたら）
      - 建築詳細
      - 近くの建築
      - アチーブメント
    - 右下に現在地ボタンをつける。
  - 上記の各種情報部分を除いたマップ部分の中心にユーザーの現在地を表示する
  - 近くの建築部分で指定した範囲の円をユーザーを中心にして重ねる
- サイドパネル
  - サイドパネル全体
    - 幅：画面全体の30%
    - 高さ：要素を並べ切れる高さ
      - 画面の高さからはみ出る場合、スクロールできるようにする
  - 建築詳細
    - タイトル・設計者表記の横に「ルートを表示」をつける
    - 下部にタグを表示
      - ない場合はタグ表示部分を省略する
  - 近くの建築
    - リストで表示
    - それぞれの表示の右端に「ルートを表示」をつける
  - アチーブメント
    - 獲得したアチーブメントの個数を表記する
    - 獲得したものをリストで表示
      - 表示を畳めるようにする
  - 配色
    - `https://map.shinkenchiku.online/`をベースとする
    - 青色
      - サイトタイトル
      - 項目見出し
      - 建築情報部分の作品タイトル
    - 黒
      - 上記以外の要素
    - 近くの建築
      - マウスオーバーすると青地に白文字で表示する

### スマートフォン/タブレット

- PCでの実装を基本的に踏襲する
- マップを画面全体に展開する
- 情報画面
  - マップ上に重なるように配置
  - 画面下部にプルして展開できるタブを作る
    - 画面縦幅の1/5程度を常時表示する
    - 表示されている際は、プル可能な状態を示すバーをプル画面の上部に配置
    - ピンを選択した場合はタイトルと画像のみ表示
      - ピンを選択した時点ではプル画面は展開しない
  - 下部画面を展開すると情報を閲覧できる
    - 建築情報全体
    - 画面上部をタッチすると画面が展開・収納される
  - 近くの建物・アチーブメントはボタン化して左上に並べる
    - ボタンを押すと左側にスライドして現れる
      - スライドしているモーションをつける。
    - 現れたページの右上に「×」マークを表示し、それを押すと右側にスライドして消える
  - 「現在地」ボタンはマップ部分の右下に配置

## 挙動

- マップ
  - 現在地ボタンを押すとユーザーの現在地がマップの中心に来るようにする。
  - それ以外の操作では現在地に移動させないようにする。
- 建築詳細
  - タグを押すと同じタグの建築のみがマップにピンされる
  - 上記の表示の際に一旦引きでマップが表示されるようにする
- 建築詳細/近くの建築
  - 右端の「ルートを表示」ボタンをクリックすると現在地からのルートを表示する
- ピン、建築を指定した際の挙動
  - 近くの建築リストの中の建築をクリックすると、その建物のピンにマップの中心が移り、少しズームする
  - 「ルート表示」を押した際、ルートの全体が出るようにマップを引き、マップの中心に表示されるように位置を動かす。
    - 位置の調整はピンの位置の表示の時のみ
    - ピンの選択を行った後、マップを動かして次のピンを選択するまではそのような挙動をしない
  - 選択されたピンやルートの到着地点のピンの色を変える
- マップ上のピンのある建物に接近する
  - 技術限界(5m)で近接した時に確認のポップアップを表示
  - 表示されたポップアップの「訪問した」ボタンを押すとアチーブメントに建物が追加される
- 近くの建築
  - マウスオーバーすると青地に白文字で表示する
- アチーブメント
  - 獲得したものをリストで表現。
  - ボタンで畳めるようにする。

## 取り扱い情報

- JSON
  - `./data`に配置
  - `init.json`
    - ピンを落とすデータを想定して作成
    - タイトル・設計者・経緯度
  - `main.json`
    - 建築ごとのページなどの表示を想定して作成
    - 詳細情報全て
    - `builduse`と`tag`は配列でデータを保持。
      - 建築用途：建築用途の表示に使用。複数の用途を同時に持つこともある。
      - タグ（`tags`）：タグ検索・表示などに使用する想定。
  - `photo.json`
    - タイトル・設計者とともにGCSに格納された画像のリンクを格納している
- 画像
  - `photo.json`で参照できるGCSのリンクを使用する
